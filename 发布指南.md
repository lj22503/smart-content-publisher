# GitHub发布指南

## 📦 完整发布流程

### 第一步：创建GitHub仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `smart-content-publisher` (推荐)
   - **Description**: `智能内容分发助手 - 支持微信公众号、飞书、小红书的自动内容分发工具`
   - **选择** Public (公开) 或 Private (私有)
   - **不要**勾选 "Add a README file" (我们已经有了)
   - 点击 **Create repository**

### 第二步：推送代码到GitHub

在项目目录中执行以下命令：

```bash
# 进入项目目录
cd "C:\Users\lj225\Documents\Obsidian Vault\fund_investment_course\04_tools\smart_publisher\v3_mvp"

# 添加远程仓库 (将 YOUR_USERNAME 替换为你的GitHub用户名)
git remote add origin https://github.com/YOUR_USERNAME/smart-content-publisher.git

# 推送代码到main分支
git branch -M main
git push -u origin main
```

**注意**: 首次推送可能会要求输入GitHub用户名和密码，如果使用双因素认证，需要使用[个人访问令牌(PAT)](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens)。

### 第三步：验证发布

推送成功后，访问你的仓库：
```
https://github.com/YOUR_USERNAME/smart-content-publisher
```

应该能看到所有代码文件。

### 第四步：发布第一个版本

```bash
# 创建标签
git tag -a v3.0.0 -m "Release version 3.0.0 - Initial MVP"

# 推送标签到GitHub
git push origin v3.0.0
```

推送标签后，GitHub Actions会自动构建Windows和Linux版本，并创建Release。

## 🚀 使用方式

### 方式一：直接运行Python代码（开发/测试）

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 安装Playwright浏览器
playwright install chromium

# 3. 运行测试
python test_integration.py

# 4. 启动应用
streamlit run main.py
# 或者使用启动脚本
启动.bat
```

### 方式二：使用打包的可执行文件（最终用户）

#### 本地打包

```bash
# 安装打包工具
pip install pyinstaller

# 打包为目录版（推荐，启动更快）
python build_exe.py --mode onedir

# 打包为单文件版
python build_exe.py --mode onefile
```

**输出位置**:
- 目录版: `dist/智能内容分发助手/`
- 单文件版: `dist/智能内容分发助手.exe`

#### 使用GitHub Actions构建

1. 推送代码到GitHub后
2. 进入仓库的 **Actions** 标签
3. 查看 **Build Executable** 工作流状态
4. 构建完成后，在工作流运行页面下载构建产物

## 🔧 应用配置指南

### 微信公众号配置

1. **获取API凭证**:
   - 登录 [微信公众平台](https://mp.weixin.qq.com)
   - 开发 → 基本配置 → 获取AppID和AppSecret

2. **添加IP白名单**:
   - 基本配置 → IP白名单 → 添加你的服务器IP或本地IP

3. **在应用中配置**:
   - 侧边栏 → 微信公众号 → 填入AppID和AppSecret

### 飞书配置

1. **创建应用**:
   - 登录 [飞书开放平台](https://open.feishu.cn)
   - 创建企业自建应用

2. **获取API凭证**:
   - 应用详情 → 凭证与基础信息 → 获取App ID和App Secret

3. **申请权限**:
   - 权限管理 → 搜索"云文档" → 勾选所有文档相关权限
   - 申请开通

4. **在应用中配置**:
   - 侧边栏 → 飞书 → 填入App ID和App Secret

### 小红书配置

1. **启动Chrome调试模式**:
   ```bash
   # Windows
   "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222

   # 或创建快捷方式，在目标中添加: --remote-debugging-port=9222
   ```

2. **登录小红书**:
   - 在打开的Chrome浏览器中访问 https://www.xiaohongshu.com
   - 使用小号登录（避免封号风险）

3. **连接应用**:
   - 侧边栏 → 小红书 → 点击"测试浏览器连接"

⚠️ **注意**: 小红书使用浏览器自动化，存在封号风险，强烈建议使用测试小号！

## 🔍 故障排查

### 常见问题

#### Q: 推送代码时报错"fatal: Authentication failed"
A: 需要使用GitHub个人访问令牌：
1. GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 生成新令牌，勾选所有repo权限
3. 推送时代替密码使用此令牌

#### Q: 小红书连接失败
A: 确保Chrome以调试模式启动：
1. 关闭所有Chrome窗口
2. 使用调试参数启动
3. 确认端口9222未被占用

#### Q: 微信公众号API调用失败
A: 检查：
1. IP是否在白名单中
2. AppID和AppSecret是否正确
3. 网络是否可以访问微信API服务器

#### Q: 打包时文件过大
A: Streamlit依赖较大，建议：
1. 使用目录版打包
2. 首次运行会下载一些缓存文件
