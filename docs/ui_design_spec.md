# 智能内容分发助手 - UI设计规范

## 1. 设计概述

### 1.1 设计目标
- **用户友好**: 零技术背景用户也能轻松上手
- **高效工作流**: 内容输入 → 智能改写 → 多平台预览 → 一键发布
- **视觉清晰**: 信息层次分明，操作反馈明确

### 1.2 设计原则
1. **渐进式披露**: 复杂配置隐藏在侧边栏，主界面保持简洁
2. **实时预览**: 改写结果即时可见，支持对比
3. **平台差异化**: 各平台预览呈现其特有风格
4. **错误预防**: 操作前检查配置，提供明确引导

- **默认**: bg-primary, hover 变深
- **加载中**: 显示 spinner，禁用点击

## 5. 交互动效

### 5.1 过渡动画
```css
/* 按钮悬停 */
transition: all 0.2s ease;
transform: translateY(-1px);

/* 卡片展开 */
transition: max-height 0.3s ease-in-out;

/* 内容刷新 */
animation: fadeIn 0.3s ease;

/* 成功反馈 */
animation: pulse 0.5s ease;
```

### 5.2 加载状态
- **按钮加载**: 显示 spinner，文字变为"处理中..."
- **预览加载**: 显示骨架屏 shimmer 效果
- **发布进度**: 进度条 + 步骤指示

### 5.3 反馈机制
| 场景 | 反馈类型 | 持续时间 |
|------|----------|----------|
| 改写完成 | Toast 成功提示 | 3s |
| 复制成功 | Toast + 图标动画 | 2s |
| 配置错误 | 红色边框 + 文字提示 | 持续 |
| 发布成功 | 绿色勾选 + 状态更新 | 3s |
| 连接测试 | 加载中 → 结果 | 2-5s |

## 6. 平台适配展示

### 6.1 小红书风格特征
- **视觉**: 每段以emoji开头
- **结构**: 短段落，每段不超过2行
- **语言**: 口语化，使用"集美们""种草"等词汇
- **结尾**: 互动引导 + 话题标签

### 6.2 公众号风格特征
- **视觉**: 清晰的标题层级
- **结构**: 导语 + 正文 + 结语
- **语言**: 专业但有温度
- **排版**: 适当留白，重点加粗
